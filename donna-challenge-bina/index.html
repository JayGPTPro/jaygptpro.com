🧠 שלב 1: מה זה Claude Code (ולמה הוא שונה)

בטח כבר השתמשתם ב-ChatGPT או Gemini. הם צ'אטבוטים: אתם שואלים, הם עונים, והם שוכחים. כל שיחה מתחילה מאפס.

Claude Code הוא משהו שונה לגמרי. הוא לא צ'אטבוט. הוא סביבת עבודה מלאה.

שלושה הבדלים מרכזיים:

1. טווח הגעה. הוא חי בתוך המחשב שלכם. הוא קורא את הקבצים שלכם, פותח את האפליקציות שלכם, משתמש בדפדפן שלכם. הוא מתחבר ל-Gmail, Calendar, Drive, Slack – הכלים שאתם כבר משתמשים בהם.

2. פעולה. צ'אטבוט מסביר. Claude Code עושה. אתם אומרים לו מה אתם רוצים. הוא מוצא דרך ומחזיר עבודה גמורה, לא הוראות.

3. אוטונומיה. הוא רץ לפי לוח זמנים, ברקע, בזמן שאתם בפגישה או מבלים זמן איכות עם המשפחה שלכם. העבודה לא נעצרת כשאתם עוצרים.

בקיצור: ChatGPT מדבר אליכם. Claude Code עובד בשבילכם.

לאורך האתגר הזה אנחנו עובדים רק מלשונית ה-Code. לא Chat, לא Cowork. רק Code.

💻 שלב 2: התקינו את Claude Desktop

Claude Code רץ בתוך אפליקציה שנקראת Claude Desktop. לא בדפדפן, לא בטרמינל. אפליקציה אמיתית על המחשב שלכם.

למק:

  1. לכו ל-claude.com/download
  2. הורידו את קובץ ה-.dmg
  3. פתחו אותו וגררו את Claude לתיקיית Applications שלכם

לווינדוס:

  1. התקינו קודם את Git (ראו את התיבה האדומה למטה). Claude יסרב לרוץ בלעדיו.
  2. לכו ל-claude.com/download
  3. הורידו את ההתקנה ולחצו "הבא" עד הסוף

למשתמשי Windows בלבד: Git נדרש

Claude משתמש ב-Git ברקע כדי לעקוב אחרי הקבצים שלכם וכך הוא יכול לבטל שינויים שהוא עצמו עשה. בלי Git, ‏Claude Code לא ירוץ על Windows. ‏Git חינמי ולוקח דקה להתקין אותו.

  1. לכו ל- git-scm.com/downloads/win
  2. הורידו והתקינו (הבא → הבא → סיום)
  3. ודאו ש-"Add Git to PATH" נשאר מסומן (זו ברירת המחדל)
  4. אימות: פתחו CMD (חפשו "cmd" ב"התחל") והקלידו git --version. אם אתם רואים מספר גרסה, אתם מסודרים.

משתמשי Mac: דלגו על זה. Git כבר מובנה.

לאחר ההתקנה, פתחו את Claude Desktop בפעם הראשונה. הוא יבקש מכם להתחבר או ליצור חשבון. עקבו אחר ההוראות שעל המסך.

💳 שלב 3: בחרו את התוכנית שלכם (התחילו עם Pro, שדרגו ל-Max כשתתמכרו)

השכבה החינמית לא פותחת את Claude Code. אתם צריכים תוכנית בתשלום. שלוש אפשרויות חשובות:

חינם

לא עובד עבור Claude Code. אתם צריכים תוכנית בתשלום. המינימום הוא Pro.

Pro . $20 לחודש

נקודת ההתחלה לאתגר הזה. פותח סקילים, Routines, קונקטורים.

Max . $100 לחודש

לאן שרוב האנשים מגיעים אחרי שבוע. יותר קרדיטים, פותח את Auto Mode (ראו שלב 8).

איך לשדרג:

  1. פתחו את Claude Desktop
  2. לחצו על אייקון הפרופיל שלכם (בפינה השמאלית התחתונה)
  3. בחרו ב-Upgrade או Subscribe
  4. בחרו ב-Pro (או Max אם אתם כבר יודעים שאתם רוצים את המרווח)
  5. הזינו את פרטי הכרטיס שלכם ואשרו

מילה על קרדיטים

לכל תוכנית יש תקציב קרדיטים שמתאפס בחלון מתגלגל. ככל שתשתמשו יותר ב-Claude, כך תרגישו מהר יותר את התקרה. אם Pro מתחיל להרגיש צפוף, Max נותן לכם בערך פי 5 יותר מקום. הדעה הכנה שלי: ברגע שתראו מה Claude Code באמת יכול לעשות, השדרוג משתלם מעצמו. אל תנסו לחסוך בכלי היחיד שיחסוך לכם מאות שעות.

טיפ: הגדירו את Sonnet כמודל ברירת המחדל שלכם כדי למתוח את הקרדיטים שלכם. פירוט מודלים מלא בשלב 7.

📁 שלב 4: Claude עובד עם תיקיות

זהו הקונספט החשוב ביותר להבנה. Claude Code לא עובד "באוויר". הוא מתחבר לתיקייה ספציפית במחשב שלכם.

לפני כל סשן עבודה אתם בוחרים תיקייה (קובץ ← פתח תיקייה). Claude קורא את הקבצים שבה, כותב אליה, וטוען את קובץ ה-CLAUDE.md שנמצא בתוכה.

זו הסיבה שדונה "חיה" בתיקייה ספציפית (donna-starter-kit). פתחו את התיקייה הזו ב-Claude Code ודונה תדע מי היא ומה היא יכולה לעשות. פתחו תיקייה אחרת ו-Claude לא יזהה אותה בכלל.

בקיצור:

תיקייה = פרויקט. תיקייה אחרת = פרויקט אחר. קובץ ה-CLAUDE.md שבתוך התיקייה הוא ההוראות שלכם ל-Claude עבור הפרויקט הזה.

הכלל שאני חי לפיו: תיקייה אחת לכל פרויקט. תיקייה לעסק שלי, תיקייה לכל השקת מוצר, ואחת לדונה. אל תערבבו נושאים בתוך אותה תיקייה.

שיחות נעוצות . גררו כל שיחה כדי לנעוץ אותה. שימושי לכמה שיחות חשובות שאתם חוזרים אליהן שוב ושוב. את השאר אפשר לקבץ לפי תאריך או פרויקט.

🔐 שלב 8: מצבי הרשאות (עכשיו עם Auto Mode)

בכל פעם ש-Claude רוצה לעשות משהו (לערוך קובץ, להריץ פקודה, לשלוח נתונים למקום כלשהו), הוא מבקש אישור . אלא אם כן אמרת לו שלא. חמישה מצבים שכדאי להכיר:

✋ בקש הרשאות

</> אישור עריכות אוטומטי

Claude עורך קבצים בלי לשאול, אבל עדיין עוצר לפני הרצת פקודות. פתרון ביניים יציב.

📋 מצב תכנון

Claude מתכנן לפני שהוא פועל. מציג לך את התוכנית, ממתין לאישור. טוב למשימות גדולות שאתה רוצה לתכנן מראש.

⚠️ עקיפת הרשאות

Claude רץ בלי לשאול. הכי מהיר, אבל בלי רשת ביטחון. השתמש בו רק כשאתה יודע בדיוק מה אתה עושה.

✨ Auto Mode . ברירת המחדל המומלצת שלי (חדש) Max בלבד (100$/חודש)

Claude רץ בלי לשאול, אבל מסווג נפרד בודק כל פעולה קודם וחוסם כל דבר מסוכן . פריסות לפרודקשן, נתונים רגישים שעוזבים את המחשב שלך, מחיקות המוניות, פושים כפויים לגיט. אתה מקבל מהירות ברמת Bypass עם רשת ביטחון. Anthropic הוסיפו את זה בסוף מרץ 2026 וזה מה שאני משתמש בו כל יום עכשיו. ב-Pro? תשתמשו ב-Auto accept edits במקום.

המלצה: התחל עם אישור עריכות אוטומטי ליום-יומיים כדי לבנות אמון. ברגע שאתה מרגיש בנוח, עבור ל-Auto Mode ותן ל-Claude לזרום.

ב-Pro? קראו את זה לפני שאתם בוחרים ב-Auto Mode

Auto Mode לא זמין בתוכנית Pro (ב-20$). הוא דורש Max, Team או Enterprise, ואת מודל Sonnet 4.6 / Opus 4.6+. אם אתם ב-Pro לאתגר הזה, השתמשו ב-Auto accept edits כברירת מחדל. כשאתם משדרגים ל-Max (ורוב האנשים עושים את זה), Auto Mode נפתח אוטומטית.

איפה לשנות את זה: לחצו על האייקון בפינה השמאלית התחתונה של שורת הקלט (ליד כפתור השליחה).

🟣 שלב 4: התקנת Obsidian

Obsidian זו אפליקציה חינמית שמציגה קבצי טקסט בצורה ויזואלית. אנחנו משתמשים בה כדי לראות את הזיכרון של דונה. כשדונה כותבת סיכום, מתעדת החלטה, או שומרת מידע על אנשים, Obsidian מציג את הכל בצורה מסודרת עם קשרים ביניהם.

  1. לך ל-obsidian.md והורד (חינמי, Mac/Windows/Linux)
  2. התקן כמו כל אפליקציה

בשלב הבא נחבר את Obsidian לתיקיית brain/ של ערכת ההתחלה.

🧠 שלב 5: מכירים את Obsidian (המוח של דונה)

Obsidian הוא המוח של דונה. כל מה שהיא לומדת, שומרת, וזוכרת, נמצא כאן כקבצי Markdown שאפשר לקרוא ולערוך.

מה זה Vault?

Vault = תיקייה שאובסידיאן יודע לקרוא. כשאתה פותח תיקייה באובסידיאן, הוא הופך אותה ל-Vault ומאפשר לך לראות את הקבצים כרשת מחוברת. אין פה שום קסם, זו פשוט תיקייה עם קבצי טקסט.

  1. פתח Obsidian ובחר את תיקיית brain/ כ-Vault
  2. עבור על התיקיות: Business (Goals, Promises, Decisions), People, Daily Logs, Preferences
  3. שים לב שהכל ריק. עכשיו ריק, ביום 5 מלא. כל פעם שדונה עובדת, היא כותבת לכאן.
  4. הכר את Graph View (אייקון בצד שמאל). עוד אין מה לראות, אבל כשדונה תתחיל לעבוד, תראה את הקשרים בין קבצים.

⚠️ קריטי: שנו כל Routine מ-"Ask permissions" ל-"Auto mode"

💡 טוב לדעת על Routines

המחשב שלך צריך להיות דולק, אבל הוא משלים פערים אוטומטית.

אתה לא צריך להשאיר את הלפטופ דולק כל הלילה. כשאתה מעיר אותו (או מדליק אותו), Claude Desktop מריץ את ה-slot האחרון שהוחמץ לכל Routine – אוטומטית, ללא טריגר ידני. אז אם הלפטופ שלך היה כבוי כל הלילה, תדריך הבוקר שלך יופיע כשתפתח אותו.

הזמן המתוזמן הוא משוער (stagger).

Claude Desktop מוסיף עיכוב אקראי של כמה דקות לכל Routine לטובת ביצועי השרת. אז Routine של 7:00 בבוקר עשוי לרוץ בפועל ב-7:04. ההיסט (offset) קבוע לכל Routine. מספר Routines באותו זמן ירוצו ברצף עם ה-stagger הזה.

רוצה שזה ירוץ גם כשהמחשב שלך כבוי?

אפשרות מתקדמת: Cloud Routines רצים על התשתית של Anthropic. דורש לדחוף את brain/ ל-repo ב-GitHub (הגדרה מורכבת יותר). לרוב האנשים, ה-Local routines שהגדרנו היום עובדים מצוין – הם משלימים פערים כשאתה פותח את הלפטופ. תוכל לחקור בהמשך ב- claude.ai/code/routines אם תרצה.
